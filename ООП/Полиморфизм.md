# Полиморфизм

**Полиморфизм** — один из ключевых принципов объектно-ориентированного программирования. Классическое определение, сформулированное Бьёрн Страуструп:

```«Один интерфейс — множество реализаций»```

**Полиморфизм** позволяет использовать единый интерфейс для работы с объектами разных типов, при этом конкретное поведение определяется самим объектом.

## Суть полиморфизма

Полиморфизм реализуется через:

- переопределение методов (override) в производных классах
- динамический выбор реализации во время выполнения программы

Это означает, что один и тот же вызов метода может приводить к разному поведению в зависимости от типа объекта.

## Полиморфная переменная

**Полиморфная переменная** — это переменная, которая:

- хранит ссылку на базовый тип
- но может указывать на объекты разных производных классов

Пример (концептуально):

```
Animal a = new Dog();
Animal b = new Cat();
```

Обе переменные имеют тип Animal, но поведение будет разным.

## Виды полиморфизма

**1. Ad hoc (специальный полиморфизм)**
- Реализуется через перегрузку методов (overloading)
- Методы имеют одно имя, но разную сигнатуру

Пример:

```
print(int x)
print(String s)
```

**2. Параметрический полиморфизм (Generics)**

Позволяет работать с обобщёнными типами, которые задаются при использовании.

Смысл generics:

- написание универсального кода;
- независимость от конкретных типов;
- обеспечение типобезопасности.

Пример:

```List<T>```

Где T — это параметр типа, который определяется при использовании.

**3. Полиморфизм подтипов (наследование)**

Основан на иерархии классов:

- базовый класс задаёт общий набор методов и поведения
- производные классы реализуют или переопределяют поведение

Реализуется с помощью:

- переопределения методов (override);
- виртуальных методов;
- интерфейсов.

## Почему (часто) запрещено множественное наследование?

Основная проблема — это неоднозначность (ambiguity).

Проблемы множественного наследования:
- Конфликт методов
- Если два базовых класса имеют одинаковый метод
→ непонятно, какую реализацию использовать
- Ромбовидное наследование (diamond problem)

    ```
        A
       / \
      B   C
       \ /
        D
    ```

    - Класс D наследует A дважды (через B и C)
    - Возникает дублирование и путаница
    - Сложность поддержки
    - Код становится трудно читаемым
    - Возрастает риск ошибок
