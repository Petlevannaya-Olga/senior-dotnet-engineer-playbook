## Абстракция

**Абстракция** — это процесс выделения существенных характеристик объекта и игнорирования несущественных деталей. Она позволяет скрыть внутреннюю реализацию и сосредоточиться на том, *что* делает объект, а не *как* именно это реализовано.

Основная идея абстракции — представить объект через минимально необходимый набор свойств и методов, достаточный для решения конкретной задачи.

### Зачем нужна абстракция

- уменьшает сложность системы;
- упрощает понимание и использование кода;
- позволяет работать с объектами на более высоком уровне;
- изолирует изменения реализации от остальной части программы.

### Как реализуется

Абстракция в ООП реализуется не одной конкретной конструкцией, а набором механизмов, которые позволяют скрыть детали реализации и оставить только необходимый интерфейс для работы с объектом.

**1. Абстрактные классы**

Абстрактный класс задаёт общую модель поведения, но может не содержать полной реализации.

- нельзя создать экземпляр напрямую;
- может содержать как реализованные методы, так и абстрактные (без реализации);
- используется как базовый класс для других.

Пример:

``` CSharp
abstract class Animal
{
    public abstract void MakeSound(); // нет реализации

    public void Sleep()
    {
        Console.WriteLine("Sleeping...");
    }
}
```

Здесь мы описали *что должен уметь делать объект*, но не указали, *как именно* он это делает.

**2. Интерфейсы**

**Интерфейс** — это более строгая форма абстракции. Он описывает только поведение без реализации.

- задаёт контракт (набор методов);
- класс обязан реализовать все методы интерфейса;
- позволяет работать с разными объектами одинаково.

Пример:

``` CSharp
interface IFlyable
{
    void Fly();
}
```

Любой класс, реализующий этот интерфейс, обязан определить метод Fly().

**3. Инкапсуляция (как часть абстракции)**

Абстракция тесно связана с инкапсуляцией:

- скрываются внутренние поля (private);
- доступ к ним осуществляется через методы (геттеры/сеттеры);
- пользователь объекта не видит внутреннюю логику.

Пример:

``` CSharp
class BankAccount
{
    private double balance;

    public void Deposit(double amount)
    {
        if (amount > 0)
        {
            balance += amount;
        }
    }
}
```

Пользователь не знает, как именно хранится баланс — он работает через методы.

**4. Разделение интерфейса и реализации**

**Главная идея абстракции:** отделить что делает объект от как он это делает. 

Пользователь работает с понятным набором методов ```car.Start();``` но не знает, какие процессы происходят внутри.

## Итог

Абстракция в ООП реализуется через:

- абстрактные классы;
- интерфейсы;
- сокрытие данных (инкапсуляцию);
- чёткое разделение интерфейса и реализации.

Она позволяет уменьшить сложность системы и работать с объектами на более высоком уровне, не погружаясь в детали их внутреннего устройства.
